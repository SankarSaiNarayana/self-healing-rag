How to showcase “self-healing” in a live demo
  • Vector store (Chroma or Qdrant) holds your document index (embeddings + text chunks). That is the “database” for retrieval. Self-healing is not a separate DB—it is extra logic that runs on each question.
  • Heal 1 — retrieval: point at Retrieval retries in the trace panel. If it is above zero, the pipeline widened search after the first pass looked weak.
  • Heal 2 — verification: point at Verification retries and Claim checks. If verification retries are above zero, it re-fetched evidence and regenerated when too many answer sentences were not supported by the retrieved text.
  • Grounding: in the JSON below, open claims to show each sentence linked to a source chunk (or flagged).
